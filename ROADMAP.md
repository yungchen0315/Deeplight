# DEEPLIGHT 後續路線圖

本文件整理 Claude Fable 5 撰寫的完整企劃（Phase 1～3），Phase 1 已於本次迭代（v1.1）實作完成，Phase 2／3 為後續工作紀錄，供下一次迭代或外部協作者接手。

## Phase 1 — 已完成（v1.1）

- 存檔 schema v2（settings／tutorial／daily／boostUntil／tapLureProgress／stats 擴充），沿用既有 localStorage key 就地遷移。
- 全程式合成 Web Audio 音效（12 種事件音）＋深度反應式環境音（低頻 drone＋噪音水床＋定期聲納 ping）。
- 畫面回饋層 `src/ui/fx.js`：數字彈出、像素粒子爆發、畫面震動、按鈕彈跳，全部尊重「減少動態」設定。
- 點擊採光改為「目前每秒產量的固定比例（有下限）」，任何階段點擊都有意義；新增誘光進度條，累積 60 次點擊強制刷新一隻路過生物。
- 平衡調整：壓載起始速度／成本、壓載最高等級 8→12、模組升級節點延伸到 100/200 持有、暮光帶錨點成本下修。
- 新增批量購買 x1／x10／xMax（模組頁）。
- 珍珠加護：消耗珍珠換取限時全產量倍率，或直接兌換數小時產量，讓珍珠不再是死貨幣。
- 深淵帶（4000–11000m）完整內容：2 種新模組、4 種新生物、4 節新研究、4 項新重構、6 個新成就，並把「即將開放」的預告牌往後推到新的第 5 海域（海溝幽域）。
- 設定彈窗（頂欄齒輪按鈕）：音效／環境音／減少動態／數字格式切換、存檔匯出匯入、重看教學、重置存檔。
- 輕量新手引導：首次進入潛航畫面顯示 4 頁式歡迎彈窗說明核心迴圈（點擊採光／模組／生物／轉生），可在設定重看。
- 每日簽到＋連續天數（7 天循環獎勵：樣本／螢光／珍珠／稀有生物擇一），底部導覽「海面」分頁有未領取紅點提示。
- PWA：`manifest.webmanifest`＋手繪 SVG 圖示（一般／maskable）＋`sw.js`（cache-first，版本化快取），可從行動裝置瀏覽器加到主畫面、離線遊玩。
- `privacy.html`：隱私權政策靜態頁（中英雙語），對應「完全離線、不收集任何資料」的實際行為。

## Phase 2 — 近期規劃（尚未實作）

1. **第二層轉生「深淵協約」**：`prestigeCount ≥ 5 且 maxDepthEver ≥ 11000` 解鎖，轉生取得「深淵印記」＝`floor(sqrt(累積壓力核心/20))`，重置核心與重構，開啟 3 分支 × 4 階＝12 節的印記樹（產量／下潛／離線與生物三大分支）。
2. **每日任務**：依日期決定性種出 3 項任務（10 個範本池），完成給樣本，全部完成加碼 1 珍珠。
3. **金燈魚事件**：每 10–20 分鐘主動遊玩觸發一次的稀有快速生物，點擊後二選一強力短效增益，是同類遊戲中最強的主動回訪誘因。
4. **圖鑑星級**：目擊次數達 1/5/15/40/100 次分別升星，星級決定圖鑑加成（取代目前扁平 +2%），延伸長期收集目標。
5. **第 5 海域「海溝幽域」**（11000–20000m）：2 新模組、4 新生物、4 節新研究，背景改用 CSS 濾鏡（hue-rotate）而非新增色盤顏色（維持 20 色上限）。
6. **週末活動**：本地日期判斷週六日，生物出現間隔 ×0.5，畫面顯示活動橫幅，純前端不需伺服器。
7. **自動化重構**（印記樹解鎖）：自動購買最便宜模組、自動收集生物、自動通過閘門。

## Phase 3 — 上架 Google Play 準備（尚未實作，需要人工帳號/金鑰）

實際送審與上架涉及只有專案擁有者能完成的步驟（開發者帳號、簽名金鑰、店面素材審核），這裡記錄可執行的具體流程：

1. **封裝方式：TWA（Trusted Web Activity）＋ Bubblewrap**。理由：本專案已經是零建置流程的靜態站台＋Service Worker，TWA 可以直接沿用同一份程式碼，不需要另外維護一份 App 專案（Capacitor 為備案，若未來需要「首次啟動不need網路」的離線安裝需求再評估）。
   - 先把靜態站台部署到一個 HTTPS 網域（GitHub Pages 即可）。
   - 執行 `npx @bubblewrap/cli init --manifest https://<你的網域>/manifest.webmanifest` 產生 `android/` 專案（需要本機安裝 Java + Android SDK，這步驟需要在你自己的機器上跑）。
   - 在網域的 `/.well-known/assetlinks.json` 放上用你的正式簽名金鑰算出的 SHA-256 指紋，完成 TWA 與網站的雙向驗證。
   - Service Worker 的 `CACHE_NAME` 版本號與 App 的 `versionCode` / `versionName` 建議手動保持同步遞增。
2. **圖示／截圖素材**：Google Play 要求 PNG（不是 SVG）。可以另外寫一個小工具頁面把 `icons/icon.svg`／`icon-maskable.svg` 畫到 canvas 匯出成 512×512 PNG，以及 1024×500 的 feature graphic、1080×1920 的截圖（截取教學畫面／潛航畫面／圖鑑／轉生畫面）。這個匯出工具還沒做，屬於 Phase 3 待辦。
3. **隱私權政策**：`privacy.html` 已經寫好，部署後把網址填進 Play Console 的隱私權政策欄位；Data Safety 表單可以直接勾選「不收集任何資料、不分享任何資料」。
4. **商店文案方向**：強調「100% 離線可玩、無廣告、無內購、純手工像素深海生物圖鑑、程式生成音效」；分類建議休閒／放置；內容分級問卷可勾選「所有年齡」（無暴力、無內購誘導）。
5. **發版衛生**：Service Worker 偵測到新版本時，於 UI 提示「有新版本，點擊更新」並呼叫 `skipWaiting()`＋重新整理；上架前建議跑過：全新安裝、舊版存檔遷移、離線冷啟動、48 小時離線結算 這幾種情境。
