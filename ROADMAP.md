# DEEPLIGHT 後續路線圖

本文件整理 Claude Fable 5 撰寫的完整企劃（Phase 1～3）與後續自主擴充。Phase 1／2／4 已全部實作完成（v1.1→v1.4），並持續進行嚴格 debug 與便利性擴充；Phase 3（實際上架）仍需要專案擁有者的帳號/金鑰才能繼續。

## Phase 1 — 已完成（v1.1）

- 存檔 schema（settings／tutorial／daily／boostUntil／tapLureProgress／stats 擴充），沿用既有 localStorage key 就地遷移。
- 全程式合成 Web Audio 音效（12 種事件音）＋深度反應式環境音（低頻 drone＋噪音水床＋定期聲納 ping）。
- 畫面回饋層 `src/ui/fx.js`：數字彈出、像素粒子爆發、畫面震動、按鈕彈跳，全部尊重「減少動態」設定。
- 點擊採光改為「目前每秒產量的固定比例（有下限）」，任何階段點擊都有意義；新增誘光進度條，累積 60 次點擊強制刷新一隻路過生物。
- 平衡調整：壓載起始速度／成本、壓載最高等級、模組升級節點延伸到 100/200 持有、暮光帶錨點成本下修。
- 批量購買 x1／x10／xMax，後續再加上「一鍵買滿」。
- 珍珠加護：消耗珍珠換取限時全產量倍率，或直接兌換數小時產量。
- 深淵帶（4000–11000m）完整內容：新模組／生物／研究／重構／成就。
- 設定彈窗（頂欄齒輪按鈕）：音效／環境音／減少動態／數字格式切換、存檔匯出匯入、重看教學、重置存檔。
- 輕量新手引導：首次進入潛航畫面顯示 4 頁式歡迎彈窗。
- 每日簽到＋連續天數（7 天循環獎勵）。
- PWA：`manifest.webmanifest`＋手繪 SVG 圖示（一般／maskable）＋`sw.js`（cache-first，版本化快取）。
- `privacy.html`：隱私權政策靜態頁（中英雙語）。

## Phase 3 — 上架 Google Play 準備（尚未實作，需要人工帳號/金鑰）

實際送審與上架涉及只有專案擁有者能完成的步驟（開發者帳號、簽名金鑰、店面素材審核），這裡記錄可執行的具體流程：

1. **封裝方式：TWA（Trusted Web Activity）＋ Bubblewrap**。理由：本專案已經是零建置流程的靜態站台＋Service Worker，TWA 可以直接沿用同一份程式碼，不需要另外維護一份 App 專案（Capacitor 為備案，若未來需要「首次啟動不need網路」的離線安裝需求再評估）。
   - 先把靜態站台部署到一個 HTTPS 網域（GitHub Pages 即可）。
   - 執行 `npx @bubblewrap/cli init --manifest https://<你的網域>/manifest.webmanifest` 產生 `android/` 專案（需要本機安裝 Java + Android SDK，這步驟需要在你自己的機器上跑）。
   - 在網域的 `/.well-known/assetlinks.json` 放上用你的正式簽名金鑰算出的 SHA-256 指紋，完成 TWA 與網站的雙向驗證。
   - Service Worker 的 `CACHE_NAME` 版本號與 App 的 `versionCode` / `versionName` 建議手動保持同步遞增。
2. **圖示／截圖素材**：Google Play 要求 PNG（不是 SVG）。可以另外寫一個小工具頁面把 `icons/icon.svg`／`icon-maskable.svg` 畫到 canvas 匯出成 512×512 PNG，以及 1024×500 的 feature graphic、1080×1920 的截圖。這個匯出工具還沒做，屬於 Phase 3 待辦（`src/ui/shareCard.js` 的做法可以直接參考複用）。
3. **隱私權政策**：`privacy.html` 已經寫好，部署後把網址填進 Play Console 的隱私權政策欄位；Data Safety 表單可以直接勾選「不收集任何資料、不分享任何資料」。
4. **商店文案方向**：強調「100% 離線可玩、無廣告、無內購、純手工像素深海生物圖鑑、程式生成音效」；分類建議休閒／放置；內容分級問卷可勾選「所有年齡」。
5. **發版衛生**：Service Worker 偵測到新版本時，於 UI 提示「有新版本，點擊更新」並呼叫 `skipWaiting()`＋重新整理；上架前建議跑過：全新安裝、舊版存檔遷移、離線冷啟動、48 小時離線結算 這幾種情境。

## Phase 4 — 已完成（v1.4）

1. **第三層轉生「永夜盟約」**（`src/systems/pactSystem.js` + `src/data/pactDefs.js`）：深淵協約 3 次以上、且史上最大深度達 35000m 解鎖，換「夜輝」點 4 分支 × 3 階的盟約樹，跟印記樹疊乘；「星海羅盤」分支獨有 3 種效果（締結時保留各分支最高階印記／每日任務自動領取／自動化效率 x2）。
2. **第 7、8、9 海域**：深海平原、熱泉海淵、裂谷深淵（35000–130000m），各 2 新模組、4 新生物（含 1 稀有）、4 節新研究；「即將開放」預告牌依序往後推。
3. **節點風味文字＋詳細彈窗**：模組、生物、研究、重構、印記、盟約節點全部補上 lore，並可點名稱彈出詳細卡片（`moduleDetailModal.js`／`bestiaryDetailModal.js`／`nodeDetailModal.js` 共用同一套設計語言）。
4. **便利性擴充**：自動化開關（可個別暫停已解鎖的自動採買/過閘門/收集/採光/領任務）、事件紀錄時間軸、存檔備份提醒、轉生試算面板、每日任務範本擴充到 20 種、固定日期季節活動（疊加在週末活動之上）。
5. **可及性與嚴格 debug 補強**：文字大小三段調整、色盲友善的非純色狀態提示（閘門就緒／目前選中數量）、把 `sprites.js` 文件裡「支援 2 幀動畫」的舊有敘述真正實作出來（少數路過生物有擺尾動畫，尊重減少動態設定）。

## Phase 5 — 已完成（v1.5）

1. **平衡性重新計算**：用一套模擬工具（含研究樹/模組升級/壓載/生物收益）重新校正第 4～8 海域的過閘門成本，把原本忽快忽慢的「時間到下一海域」曲線抹平成穩定的 1.3～2.5 倍區間；手動點擊加上 0.3 秒最短間隔、生物爆發獎勵時長從 30 秒收斂到 20 秒，避免數值提早暴衝。
2. **操作邏輯透明化**：新增「手動點擊：確切公式」「自動生產：確切公式」兩節說明（`compendiumDefs.js`），並在「模組」頁補上「目前產量加成」面板，修正單體產量顯示跟頂欄總產量對不上的問題。
3. **聲納脈衝（主動技能）**：潛航畫面新增可主動觸發的稀有生物強制刷新技能，跟誘光進度條（點擊次數觸發）互相獨立，讓生物出現節奏不再是純被動等待。
4. **除bug**：修正一般轉生漏套用「自動研究」印記效果、錨點閘門可能卡進 comingSoon 預告牌、`zoneForDepth` 在錨點深度邊界誤判下一海域、非單調任務基準值在轉生後卡住等問題。
5. **第 9 海域「先驅遺跡」**（130000–210000m）：2 新模組（遺跡節點／虛空之心）、4 新生物（含 1 稀有：先驅之影）、4 節新研究，「即將開放」預告牌推到第 10 海域；潛航日誌新增 3 篇條目，呼應深淵訊號殘片終章埋下的伏筆。
6. **版本更新提示**：Service Worker 偵測到新版本、實際取得控制權時彈出確認彈窗提示重新整理套用，只在分頁載入時就已有舊 SW 在控制的情況下才會觸發，避免玩家第一次安裝被誤判成「有更新」。

## Phase 6 — 已完成（v1.6）

1. **第 10 海域「回音迴廊」**（210000–340000m）：2 新模組（回音尖塔／起源透鏡）、4 新生物（含 1 稀有：起源之眼）、4 節新研究，「即將開放」預告牌推到第 11 海域；過閘門成本一樣用模擬工具重新校正（z8→z9 約 30.9h、z9→z10 約 52.3h，比例 1.69x）。潛航日誌新增 3 篇條目，呼應深淵訊號殘片「深淵裡至少還有其他三處同樣訊號源」的伏筆——這條迴廊看來就是通往那些地方的路。
2. **獨立審查除bug**：兩輪跳出當下開發脈絡的獨立審查（一輪針對新內容、一輪針對全專案）分別揪出：存檔匯入後 `location.reload()` 觸發的卸載自動存檔可能用記憶體裡的舊進度蓋掉剛匯入的內容；一個成就的名稱／敘事明明綁定第 9 海域，實際判定條件卻不需要玩家踏進去就能解鎖；一篇日誌條目的內文斷言玩家深度，但實際解鎖條件跟深度完全無關、可能在深度遠遠不足時就先觸發。三個都已修正並補上針對性測試。
3. **QoL 補強**：補上永夜盟約達標的情境提示（比照既有的深淵協約提示）；先驅遺跡的環境音頻率補齊（原本因為陣列缺一格、悄悄複用裂谷深淵的音色）；補上聲納脈衝的情境提示（這個機制從一開始就沒有解鎖門檻、也完全沒有任何當下說明）；新增印記樹／盟約樹點滿的潛航日誌條目；新增萬聖夜／聖誕節兩個季節活動，填補全年只有新年跟世界海洋日兩檔活動的空窗期；歡迎彈窗的每一頁補上對應底部導覽按鈕的提示光框（尊重減少動態效果設定）。
