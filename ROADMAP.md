## Phase 3 — 上架 Google Play 準備（尚未實作，需要人工帳號/金鑰）

實際送審與上架涉及只有專案擁有者能完成的步驟（開發者帳號、簽名金鑰、店面素材審核），這裡記錄可執行的具體流程：

1. **封裝方式：TWA（Trusted Web Activity）＋ Bubblewrap**。理由：本專案已經是零建置流程的靜態站台＋Service Worker，TWA 可以直接沿用同一份程式碼，不需要另外維護一份 App 專案（Capacitor 為備案，若未來需要「首次啟動不need網路」的離線安裝需求再評估）。
   - 先把靜態站台部署到一個 HTTPS 網域（GitHub Pages 即可）。
   - 執行 `npx @bubblewrap/cli init --manifest https://<你的網域>/manifest.webmanifest` 產生 `android/` 專案（需要本機安裝 Java + Android SDK，這步驟需要在你自己的機器上跑）。
   - 在網域的 `/.well-known/assetlinks.json` 放上用你的正式簽名金鑰算出的 SHA-256 指紋，完成 TWA 與網站的雙向驗證。
   - Service Worker 的 `CACHE_NAME` 版本號與 App 的 `versionCode` / `versionName` 建議手動保持同步遞增。
2. **圖示／截圖素材**：Google Play 要求 PNG（不是 SVG）。可以另外寫一個小工具頁面把 `icons/icon.svg`／`icon-maskable.svg` 畫到 canvas 匯出成 512×512 PNG，以及 1024×500 的 feature graphic、1080×1920 的截圖。這個匯出工具還沒做，屬於 Phase 3 待辦（`src/ui/shareCard.js` 的做法可以直接參考複用）。
3. **隱私權政策**：`privacy.html` 已經寫好，部署後把網址填進 Play Console 的隱私權政策欄位；Data Safety 表單可以直接勾選「不收集任何資料、不分享任何資料」。
4. **商店文案方向**：強調「100% 離線可玩、無廣告、無內購、純手工像素深海生物圖鑑、程式生成音效」；分類建議休閒／放置；內容分級問卷可勾選「所有年齡」。
5. **發版衛生**：Service Worker 偵測到新版本時，於 UI 提示「有新版本，點擊更新」並呼叫 `skipWaiting()`＋重新整理；上架前建議跑過：全新安裝、舊版存檔遷移、離線冷啟動、48 小時離線結算 這幾種情境。

2. **成就獎勵改為永久加成**：原本成就只掉深海珍珠，但珍珠終究會被拿去換成限時加護、用完就沒了——實際體感上成就的獎勵是「一時的」。現在每解鎖 1 個成就額外疊加全螢光產量永久 +0.5%（`ACHIEVEMENT_PROD_PCT`，跟圖鑑星級同一套邏輯），不受任何轉生重置，「海面」頁成就面板會即時顯示目前的加成百分比；珍珠獎勵維持不變。
3. **順手修正**：成就面板的珍珠數字原本沒有走 `formatNum` 分級顯示，跟先前修過的其他六處數字顯示是同一類問題，一併修正。

## Phase 11 — 已完成（v1.11）

## Phase 12 — 已完成（v1.12）：引導與回訪體驗

以「讓遊戲有成為膾炙人口手遊的潛力」為目標，這輪不堆數值，改補放置遊戲最影響留存的兩個環節——新手不流失、回訪有明確下一步。

1. **航路目標**（`src/systems/voyageGoalSystem.js` + 潛航畫面底部常駐橫幅）：放置遊戲新玩家流失率最高的時刻是「不知道接下來要幹嘛」，原本的情境提示（hintSystem）只在第一次符合條件時推一則一次性 toast，過了就沒有常駐指引。新增依存檔狀態算出「接下來最該做的一件事」的常駐橫幅，可測量目標附進度條，優先序涵蓋整條成長曲線（買第一個模組 → 收集生物 → 完成研究 → 通過閘門 → 重返海面 →〔已在軌道上時〕締結協約/盟約 → 長線常駐的「潛到下一片海域」→ 看完所有內容後的收束語）。純導引層，不發任何新獎勵，刻意避免跟新手任務／每日任務／里程碑／成就重疊。
2. **獨立審查抓到的 bug**：轉生會清空 `save.modules`／`save.research`，但航路目標的「買第一個模組」「完成第一個研究」兩分支只看當輪狀態，導致老玩家每次轉生後被丟回新手目標、蓋掉長線目標，連帶讓橫幅的「達成」綠色閃光在其實倒退時誤觸發。修正：這兩分支加 `prestigeCount < 1` 守衛，只在第一輪當引導。
3. **離線回訪報告改版**：標題改「歡迎回來」、螢光獲得高亮、按鈕改「繼續下潛」，並在報告底部重申目前的航路目標，讓玩家一回來就有明確下一步。`showOfflineReport` 新增 save 參數並保留舊呼叫方式往後相容。
4. **統計面板補強＋版本字串修正**：「海面」頁統計新增史上最大深度（含海域名）／里程碑領取數／永夜盟約次數／盟約樹節點數，累計數字改走 `formatNum`；修正 surfaceScreen 還寫 v1.6、settingsModal 寫 v1.11 的過時版本字串，統一為 v1.12。
